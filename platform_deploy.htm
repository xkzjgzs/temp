新能源平台安装部署文档

一、安装操作系统

（其中安装时在软件选择界面 选择应用开发，附加勾选开发工具！！！）

二、配置系统相关

1.    修改主机名

修改/etc/sysconfig/network文件，修改主机名。

 

2.    修改端口映射

修改/etc/hosts文件，修改端口映射，必须与主机名一致。

 

3.关闭防火墙

service firewalld  stop

chkconfig firewalld off

 

4.修改资源限制

vi /etc/security/limits.conf

追加如下内容

p2000   soft   noproc   20000

p2000   hard   noproc   20000

p2000   soft   nofile   20000

p2000   hard   nofile   20000

root   soft   noproc   65536

root   hard   noproc   65536

root   soft   nofile   65536

root   hard   nofile   65536

 

5.关闭selinux

vi /etc/selinux/config

将SELINUX值修改为disabled

SELINUX=disabled

重启系统

reboot

 

三、数据库安装及模板导入

1.数据库安装（只看安装部分即可）

2.导入数据库模板

使用root用户登录，并进入/home下，将模板库psgsm1218.dmp上传至/home目录下，执行：

cd /home/dmdbms/bin

./dimp  SYSDBA/SYSDBA@本地ip FILE=/home/psgsm1218.dmp  log=/home/psgsm.log

 

3.修改数据库相关表配置

CONF_BROKER，ip修改为本机ip

 

CONF_KAFKAPLATFORM，ip修改为本机ip

 

CONF_REDIS，ip修改为本机ip

 

 

四、平台部署

root用户进行如下操作

1.创建用户

useradd  p2000

修改用户密码

passwd  p2000

 

2.上传程序

将xny-kylin3.3-20221111.tar.gz包放置在/home/中后解压

cd /home/

tar zxvf xny-kylin3.3-20221111.tar.gz

 

将enc_send.conf上传到根目录下的 /etc 中

将p3spd文件上传到/etc/init.d/下，并赋予执行权限（root用户）

chmod +x /etc/init.d/p3spd

3.修改环境变量

vim /etc/profile

在最后追加如下内容：

export PSGSM_HOME=/home/p2000/PSSSP

刷新变量

source /etc/profile

4.挂载加密卡驱动

加密卡驱动需要根据服务器的GCC和内核版本来进行编译,不同的服务器之间不可通用,填写加密卡编译申请单后发给科东公司

将加密卡安装包导入要安装的服务器，使用tar -zxvf xxxxx.tar.gz 来解压加密卡安装包


解压后进入文件夹

执行./install.sh进行安装，等待安装结束

进入安装目录cd /home/p2000/kedong/SecCard 执行ls命令可以看到该目录下的加密卡驱动程序

执行./ setcap-rocky.sh 提示成功后，再执行./ insmod-rocky

挂载后执行lsmod | grep kd 来进行查询，正常如下card_ssx06后面有2个程序，如发现显示为1，请检查enc_send程序是否启动。

 

5.配置enc_send程序证书

创建证书

在home目录下创建certs目录(使用root用户)

mkdir /home/certs

mkdir /var/psgsm/

chown p2000.p2000 /home/certs

chown p2000.p2000 /var/psgsm/

 

使用p2000用户：

su - p2000

生成sm2证书

cd /home/p2000/PSSSP/bin

./enc_send exportsm2

会在/home/certs/目录下生成 ENCCARDSM2.csr

用ENCCARDSM2.csr请求文件生成ENCCARDSM2.cer证书，并复制到certs目录下，改名

mv ENCCARDSM2.cer 127.0.0.2.cer

 

生成rsa证书

cd /home/p2000/PSSSP/bin

./enc_send exportcsr

会在/home/certs/目录下生成device.csr

用device.csr请求文件生成device.cer证书，并复制到certs目录下，改名

mv device.cer 127.0.0.1.cer

 

证书签发一般由上级单位签发。

6.申请平台授权

使用root用户执行：

java -jar /home/p2000/PSSSP/uiserver/datacenter/datacenter.jar --code

将申请码发给科东公司申请授权，将授权文件authorization.lic使用p2000用户放到/home/p2000/PSSSP/目录中

 

7.启动平台程序

1）root用户执行

setsid /home/p2000/root_start/guard_dmi.sh &

 

2）切换p2000用户执行

cd ~

./start-server.sh

/etc/init.d/p3spd status

查看程序运行状态

 

8.检查程序运行正常后配置开机启动

vi /etc/rc.d/rc.local 追加如下内容：

cd /home/p2000/kedong/SecCard;./insmod-rocky

setsid /home/p2000/root_start/guard_dmi.sh &

在文件最后增加echo 1 >/proc/gwm/enc_bypass

增加脚本的执行权限

chmod +x /etc/rc.d/rc.local

 

五、补充文档

以下操作请在部署完成后资产添加前进行

1.root用户执行

cd /home/p2000/PSSSP/bin

setcap "all=ep" enc_send

setcap "all=ep" agentsyslog

2.修改PSSSP.ini配置文件

p2000用户修改/home/p2000/PSSSP/conf/PSSSP.ini配置文件在

SECURITY_AREA0=0.0.0.0

SECURITY_AREA1=0.0.0.0

SECURITY_AREA3=0.0.0.0

SECURITY_AREA4=0.0.0.0

后面添加：

SECURITY_AREA5=0.0.0.0

 

3.数据库操作

六、常见问题

1.    如果涉及enc_send提权问题，root下执行

setcap "cap_net_admin,cap_net_raw,cap_sys_rawio,cap_net_bind_service,cap_dac_read_search+ep"   /home/p2000/PSSSP/bin/enc_send

 

2.    界面不显示数据网关机配置，root用户，登录数据库，执行

UPDATE USER_ROLEFUNCTION SET  ACCESS='0'  WHERE FUNCTIONID = 'equipmentBaseSet';

 

3.    有的个别程序启动不了

1）停掉所有程序

/etc/init.d/p3spd stop

2）然后进入PSSSP/bin目录

cd /home/p2000/PSSSP/bin/

3）将所有*.lock文件删除

4）启动程序，观察程序运行状况（多刷新几次）

/etc/init.d/p3spd start&&/etc/init.d/p3spd status
